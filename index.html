Distributed Hash-Based Threshold Signatures

No single trustee holds the full key. Signing requires all k trustees to complete a two-round protocol — quantum-resistant, no secret reconstruction, standard verification.

128 security bits 288 B signature size 73 tests passing 5.9× XOR optimisation Python · stdlib only
paper Kelsey, Lang & Lucks (2024) — Turning Hash-Based Signatures into Distributed Signatures and Threshold Signatures · IACR CiC 2/2/24
how it's built

Layer architecture

PRF Distributed HBS

Two-round signing protocol. Each trustee contributes a PRF share; the aggregator combines them without learning any individual key.

top layer

Stateful Hash-Based Signatures

Manages a Merkle tree of one-time keys, tracks which have been used, and handles authentication paths for the non-distributed baseline.

state layer

Merkle Tree

Commits to the full set of OTS public keys. Construct, path, verify — O(D), O(D + log D), O(log D) respectively.

tree layer

Winternitz OTS

One-time signature built from hash chains. W=8 gives 288 B signatures with 256-step chains. Lazy variant stores only 18-byte seeds.

OTS layer

Tweakable Hash (SHA-256 / BLAKE2b)

Swap backends in one line. BLAKE2b gives ~1.18× speedup on short inputs — same security, lower per-call overhead.

what we measured

Optimisations

01 · integer XOR

_xor_many fast path

Replaces byte-by-byte XOR with a single integer operation. int.from_bytes + ^ is one C-level call instead of 16 interpreter iterations.

5.9× faster · 6.46 μs → 1.10 μs
02 · hash backend

BLAKE2b vs SHA-256

One-line swap in utils.py. BLAKE2b has lower per-call overhead on short inputs — same 128-bit security target, measurably faster keygen.

1.18× faster keygen · 3.70 → 3.14 ms
03 · lazy chains

GenLazy / SignLazy

Eager stores 18×256×16 = 73 728 B per key. Lazy stores only 18 seeds (288 B) and recomputes on demand. Right tradeoff for constrained devices.

256× smaller storage

Complexity

Scheme Operation Complexity Note
Winternitz OTS Gen O(A · CHAIN_LEN) Generates all hash chains
Sign O(A + C) Baseline one-time signing
Verify O((A + C) · CHAIN_LEN) Reconstructs public chain tips
Merkle Tree MT_Construct O(D) Builds the full auth tree
MT_MakePath O(D + log D) Constructs then extracts path
MT_Verify O(log D) Verifies one authentication path
Stateful HBS StatefulSign O(OTS sign + log D) Non-distributed baseline
Distributed HBS ShardSetup O(D · k · (A+C) · CHAIN_LEN) Collects PRF contributions per trustee
AggregatorSign O(k · (A+C) · CHAIN_LEN + k log D) Two-round PRF-based signing
AggregatorVerify O(OTS verify + log D) Same target as stateful HBS
quick start

Example

pythonend-to-end: setup → sign → verify
import os
import shard_trustee as st
import distributed_signing as ds
from utils import N_BYTES

# D = number of one-time keys, n = trustees, k = required to sign
D, n, k = 4, 2, 2
cl = [list(range(1, k + 1)) for _ in range(D)]
trustee_seeds = {t: os.urandom(N_BYTES) for t in range(1, n + 1)}

def contribution_provider(t, key_id, r, path_lens, sk_shape):
    return ds.KK_SetupContribution(trustee_seeds[t], key_id, r, path_lens, sk_shape)

# --- Setup ---
cpk, crvs, trustee_init = st.ShardSetup(D, n, cl, contribution_provider)
for t, info in trustee_init.items():
    st.TrusteeSetup(t, trustee_seeds[t], info["allowed_keyids"], info["path_lens"])

# --- Sign ---
r, path, z = st.AggregatorSign(b"hello distributed world", crvs, keyID=0)

# --- Verify (same API as non-distributed) ---
assert st.AggregatorVerify(b"hello distributed world", r, path, z)
