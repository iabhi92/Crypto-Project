01 WINTERNITZ OTS — KEYGEN · SIGN · VERIFY · FORGE

Winternitz chains computed live. Every key is unique.

hash_lms(tweak, *data) = SHA-256(tweak₀_be32 ‖ tweak₁_be32 ‖ … ‖ data)[:N_BYTES]. Keygen walks 18 chains of 255 steps each (4,590 calls). Sign looks up chain position b_all[i] — O(A+C) total. Verify continues from Z[i] for CHAIN_LEN−1−b_all[i] more steps, reconstructing the public key tip. The checksum digits prevent digit-increment forgery: increasing any message digit forces the checksum to decrease, requiring the adversary to also advance a checksum chain beyond the signed position.

02 KEY REUSE ATTACK — WHY OTS KEYS ARE ONE-TIME

Sign the same keyId twice. Watch the chain positions leak.

After observing Z1[i] at digit position b1[i] and Z2[i] at position b2[i], an adversary can compute every chain element from position min(b1[i], b2[i]) upward by forward-hashing from whichever Z has the lower index — using the same absolute step indices the keygen used. Only positions 0 … min(b1[i], b2[i])−1 remain preimage-hard. With two uniformly random messages, the expected minimum is ≈ 85, so ≈ 67% of each chain is exposed. The defence is used_keys tracking in KK_Sign1(): a trustee refuses to contribute to a second signing under the same keyId.

03 SECURITY PROPERTIES

Four properties. Each grounded in the construction.

Security reduces to second-preimage resistance of SHA-256 under the hash-based OTS proof of Buchmann et al., extended by Kelsey, Lang & Lucks to the distributed setting. No number-theoretic assumptions.

One-Time Security

A Winternitz key can sign exactly one message.

Signing at digit position b[i] exposes chain elements 0 … b[i]−1 of chain i (they are derivable from Z[i] by forward hashing). A second message with a smaller digit d' < b[i] at position i would require the adversary to produce Z'[i] at position d', which is a preimage of Z[i] under the chain function — computationally equivalent to SHA-256 second preimage.

Threshold Security

Any k−1 trustees learn nothing about the signing key.

Each trustee's PRF contribution PRF_Chain(seed_t, keyId, i, j) acts as a one-time pad over the aggregator's masked key CRV.SK[i][j]. A coalition of k−1 trustees observes a set of PRF images that are jointly uniform over the output space — the unmasked key SK[i][j] has negligible mutual information with their joint view.

Forgery Resistance

Advancing any digit forces the checksum to decrease.

The Winternitz checksum is csum = A×(CHAIN_LEN−1) − Σb[i]. Incrementing any message digit by δ decreases csum by δ, requiring the adversary to produce a chain element at a lower position for the checksum chains. That is a SHA-256 preimage — the Forge & Fail demo above demonstrates this directly: flipping one bit in Z[0] yields a reconstructed PK that does not match the stored value.

Verifier Transparency

AggregatorVerify is identical to standard OTS verification.

The verifier receives (R, Z, PATH) and calls lamport.Verify(R, Z, M, KeyID) followed by a Merkle path check — the same function used for non-distributed stateful HBS. The number of signing trustees k, the CRV structure, and the PRF contributions are invisible to the verifier. Verification cost is O(OTS verify + log D), independent of k.

04 KK_AGGREGATORSIGN — TWO-ROUND PRF PROTOCOL

k-of-n trustees. Click a trustee to mark it compromised.

Round 1: each active trustee t computes Rt = PRF_R(seed_t, keyId) and CHKt = PRF_Chk(seed_t, keyId), sends both to the aggregator. Aggregator derives CRV.R = XOR(R_base, R₀, … Rₙ) and per-trustee integrity tokens CRV.CHK[t] = XOR(PRF_Auth(seed_t, R), CHK₀, CHK₁, …). Round 2: trustee t receives R', verifies it against CHK_Prime (MAC check via KK_Auth), then contributes Zt[i] = WINTER(h, SK_share_t). Aggregator XOR-combines contributions: Z[i] = XOR(CRV.SK[i][b], Zt₀[i], Zt₁[i], …).

05 MT_CONSTRUCT — MERKLE AUTHENTICATION TREE

D=4 leaves. Internal nodes computed with real SHA-256.

Leaf i = Winternitz OTS public key for slot i, computed as hash_lms([0, i], *tips_i). Internal node at level ℓ, position p: hash_lms([3, ℓ, p], left, right). 07 _XOR_MANY() OPTIMISATION — MEASURED ON THIS MACHINE

int.from_bytes XOR dispatches once at C level. Measured live.

In CPython, int.from_bytes(v, 'big') ^ acc is a single bytecode instruction dispatched to C regardless of byte length. A byte-loop over 16 bytes dispatches 16 times. # int XOR: single C dispatch regardless of byte width
def _xor_many(values):
    if not values: return b""
    n = len(values[0])
    acc = int.from_bytes(values[0], "big")
    for v in values[1:]:
        if len(v) != n:
            raise ValueError("length mismatch")
        acc ^= int.from_bytes(v, "big")
    return acc.to_bytes(n, "big")

03 · GenLazy()

Deferred chain evaluation

Gen() precomputes all 18 × 256 chain positions at keygen, storing a 73 KB key matrix. GenLazy() stores only the 18 × 16-byte seeds (288 bytes) and walks each chain at sign time, shifting the 4,590-call cost from keygen to sign. Same PK, same signature format, same verify path.

09 SOURCE — CRITICAL PATHS

The two functions that carry all protocol complexity.

hash.py· hash_lms() — domain-separated SHA-256
def hash_lms(tweak, *data):
    h = hashlib.sha256()
    for t in tweak:
        if isinstance(t, int):
            h.update(t.to_bytes(4, "big"))   # tweak ints → big-endian 4-byte prefix
        else:
            h.update(t)
    for d in data:
        h.update(d)
    return h.digest()                          # 32 B; callers slice [:N_BYTES]

def KK_SetupContribution(seed, KeyID, R, path_lens, sk_shape):
    rows, cols = sk_shape
    return {
        "Rt":    PRF_R(seed, KeyID, N_BYTES),       # tweak (30, keyId) — masks R_base
        "CHKt":  PRF_Chk(seed, KeyID, N_BYTES),     # tweak (31, keyId) — round-2 integrity
        "Auth":  PRF_Auth(seed, KeyID, R),          # tweak (32, keyId) — binds R to seed
        "PATHt": [PRF_Path(seed, KeyID, idx, n)
                  for idx, n in enumerate(path_lens)],
        "SKt":   [[PRF_Chain(seed, KeyID, i, j, N_BYTES)
                   for j in range(cols)]      # one-time pad over CRV.SK[i][j]
                  for i in range(rows)],
    }
