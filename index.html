// 06 THREE MEASURED OPTIMISATIONS

Every speedup is benchmarked.

01 · _xor_many

Integer XOR fast path

Replaces 16-iteration byte loop with a single C-level integer op. int.from_bytes + ^ in CPython is one interpreter call vs 16. Measured over 20,000 runs with 10×16-byte operands.

Before — byte loop
6.59 μs
for b in zip(*vals):
  acc ^= b
After — int XOR
1.10 μs
int.from_bytes(v)
acc ^= ...
↑ 5.9× faster · 20 000 runs
02 · BLAKE2b

Hash backend swap

One-line change in utils.py. BLAKE2b has lower per-call overhead on short 16-byte inputs. Same 128-bit security target — just faster for the short domain inputs used throughout.

03 · GenLazy

Lazy chain evaluation

Eager Gen stores all 256 chain elements per digit (73 KB/key). GenLazy stores only 18 seeds and recomputes on demand. Same public key — dramatically less storage for constrained devices.

distributed_signing.py · _xor_many()
def _xor_many(values):
    # int XOR is ~6x faster than byte-by-byte in CPython
    if not values: return b""
    n = len(values[0])
    acc = int.from_bytes(values[0], "big")
    for v in values[1:]:
        if len(v) != n: raise ValueError("length mismatch")
        acc ^= int.from_bytes(v, "big")
    return acc.to_bytes(n, "big")

def KK_SetupContribution(seed, KeyID, R, path_lens, sk_shape):
    rows, cols = sk_shape
    return {
        "Rt":    PRF_R(seed, KeyID, N_BYTES),       # tweak (30,)
        "CHKt":  PRF_Chk(seed, KeyID, N_BYTES),     # tweak (31,)
        "Auth":  PRF_Auth(seed, KeyID, R),          # tweak (32,)
        "PATHt": [PRF_Path(seed, KeyID, idx, n)     # tweak (33,)
                  for idx, n in enumerate(path_lens)],
        "SKt":   [[PRF_Chain(seed, KeyID, i, j, N_BYTES)  # tweak (34,)
                   for j in range(cols)] for i in range(rows)],
    }
